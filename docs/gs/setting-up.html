Octoduino Documentation

Getting Started -> Setting Up

Getting ready

Thank you for trying out Octoduino! This part of the documentation will walk you through how to get started using Octoduino. If you run into any trouble, take a look at ~/docs/gs/troubleshooting.html.

Materials needed

To use Octoduino, you will need:

  • A certified device (see ~/docs/gs/certified.html)
  • A USB SD card reader
  • A Linux computer with the Python runtime
    • OR
  • A computer with git or a GUI variant installed
  • A SPI SD card reader for your device (we reccommend the official LadyAda Ethernet Shield)
  • USB cable for communicating with your device
  • FTDI chip (if needed)

With that out of the way, let's get on to installing Octoduino!

Flashing the Arduino

Before you begin, make sure your device has the Arduino bootloader installed. If your device is Arduino-certifiec then you don't need to worry about this.

Grab a copy of Octoduino from http://github.com/NETponents/octoduino/releases. If none are available, get the latest source by running git clone https://github.com/NETponents/octoduino.git inside of an empty folder.

Now open up the Arduino IDE and open ~/src/main.ino. Then go ahead and flash it to your device. If you hit an error, make sure there are no shields/wires attached other than the USB cable. Once that finishes, unplug your Arduino.

Installing Octoduino

Insert an SD card into your computer. Make sure it is completely empty. Format it to FAT32. If it asks you to name the partition, feel free to name it what ever you want.

Now open up the folder ~/bootloader. Inside it you should find several files. Copy all of the files to the root of the SD card.

First boot

Go ahead and put the SD card into your Arduino. Then plug it into your computer with the serial console running. If all goes well, you should see it start up, initialize SWAP space, and then halt. If you hit an error, or your Arduino stalls for a long time (> 2 minutes), see the troubleshooting page for help.

Some important tips

  • DO NOT remove the SD card while the Arduino is plugged into power!
  • DO NOT try to edit any .pba file. Just trust us on this one.
  • DO NOT mess with the SWAP folder.
  • DO NOT try runnning without an SD card, bad things will happen on subsequent boots.
  • DO file a bug on GitHub if you find an issue with Octoduino
